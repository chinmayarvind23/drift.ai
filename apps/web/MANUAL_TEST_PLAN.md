# Drift Manual Test Plan

Run:

```powershell
npm run dev:api
npm run dev:web
```

## Page Checks

### Scan `/`

Clean state:
- Go to `/settings`.
- Click `Wipe local data`.

Expected:
- Source is `No data yet`.
- Drift Score is `0`.
- Overspend is `$0`.
- If saved and invested is `$0`.

### Evidence `/evidence`

Import `tests/fixtures/sample-drift.csv`.

Expected:
- Evidence rows are searchable.
- Category edits immediately update the recalculated scan panel.
- Returning to `/` shows the same recalculated metrics.

### Category Detail `/category/Dining`

Expected after importing `sample-drift.csv`:
- Shows old monthly average, recent monthly average, monthly overspend, timeline, transactions.
- Shows the Pattern Lab note after one is saved for Dining.

### Pattern Lab `/insights`

Use answer:

```txt
I got a new job and felt like I deserved nicer dinners after long weeks.
```

Expected:
- Suggested tag: `Reward spending`.
- No model name or confidence is shown.
- Tag can be edited before saving.
- Saved insight appears in Plan and Report.

### Plan `/plan`

Expected after saving a Pattern Lab note:
- Recovery path references the saved behavior tag.
- Steps target repeat patterns, not one-off purchases.

### What-if `/scenario`

Set annual return to `0`.

Expected:
- If saved and invested growth is `$0`.
- Drift Score and Overspend do not change.

Set annual return to `9` and years to `20`.

Expected:
- If saved and invested growth increases.
- Drift Score and Overspend still do not change.

### Intercept `/intercept`

Expected with a drift fixture:
- `Simulate transaction` flags a repeat high-drift category.
- Marking intentional saves a decision.
- The Report shows the intercept result.

### Report `/report`

Expected:
- Executive summary.
- Financial AI review.
- Top 3 drift patterns.
- New patterns to review.
- AI behavior explanation.
- 30-day recovery path.
- Intercept result.
- Privacy note.
- Export PDF button downloads `drift-scan-report.pdf`.
- Email capture.

### Account `/account`

Expected after Supabase schema and keys:
- `Sync summary backup` shows `Summary synced. Raw transactions were not uploaded.`
- Supabase `audit_snapshots` has summary JSON only.

### Bank Sync `/connect`

Expected:
- `Sync sandbox transactions` uses Plaid sandbox only.
- No real bank account is required.
- Scan updates or shows a healthy state if no overspend exists.

### Privacy `/settings`

Expected:
- Explains local encrypted storage.
- Wipe local data resets the app.
- Does not imply raw transaction backup.

## Optional Local Qwen Through Ollama

Install Ollama and run:

```powershell
ollama pull qwen2.5:0.5b
ollama serve
```

Optional env overrides:

```txt
NEXT_PUBLIC_DRIFT_LOCAL_LLM_ENABLED=true
NEXT_PUBLIC_OLLAMA_GENERATE_URL=http://localhost:11434/api/generate
NEXT_PUBLIC_OLLAMA_MODEL=qwen2.5:0.5b
```

Expected on `/report`:
- Recovery path still appears if Ollama is off.
- If Ollama is reachable, recovery text says it was generated by local Qwen through Ollama.
